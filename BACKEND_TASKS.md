#### 2.2 Модели данных
Создать модели для хранения:
- `AIRequest` - запись о запросе к ИИ
- `AIResponse` - ответ от ИИ
- `AICache` - кэширование ответов для повторных запросов

Поля модели `AIRequest`:
- `user` - пользователь, сделавший запрос
- `request_type` - тип запроса (risk_analysis, recommendation, etc.)
- `input_data` - JSON с входными данными
- `status` - статус (pending, processing, completed, failed)
- `created_at`, `updated_at`

Поля модели `AIResponse`:
- `request` - связь с AIRequest
- `response_data` - JSON с ответом от ИИ
- `model_version` - версия модели ИИ
- `processing_time` - время обработки в миллисекундах
- `confidence_score` - уверенность модели (если применимо)
- `created_at`
