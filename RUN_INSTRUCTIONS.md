## ✅ Предварительные требования

Перед запуском проекта убедитесь, что установлено следующее ПО:

### Необходимое программное обеспечение
- **Python 3.8+** - Для Django бэкенда
- **Node.js 18+** и **npm** - Для Vue.js фронтенда
- **Git** - Для контроля версий

### Проверка установки
```bash
python --version
node --version
npm --version
```

## 🔧 Настройка окружения

### Окружение бэкенда (Django)

Перейдите в директорию бэкенда:
```bash
cd SiriusBackend
```

#### Вариант 1: Использование Pipenv (Рекомендуется)
```bash
# Установите Pipenv, если он еще не установлен
pip install pipenv

# Установите зависимости из Pipfile
pipenv install

# Активируйте виртуальное окружение
pipenv shell
```

#### Вариант 2: Использование venv
```bash
# Создайте виртуальное окружение
python -m venv venv

# Активируйте виртуальное окружение
# На Windows:
venv\Scripts\activate
# На Linux/Mac:
source venv/bin/activate

# Установите зависимости
pip install -r requirements.txt
```

### Окружение фронтенда (Vue.js)

Перейдите в директорию фронтенда:
```bash
cd SiriusFrontend/SiriusVue
```

Установите зависимости:
```bash
npm install
```

## 🐍 Настройка и запуск бэкенда

### Шаг 1: Настройка переменных окружения

Бэкенд использует настройки Django в файле [`SiriusBackend/SiriusMain/settings.py`](SiriusBackend/SiriusMain/settings.py). Ключевые настройки для проверки:

- Конфигурация базы данных (по умолчанию SQLite)
- Настройки CORS для фронтенда
- Секретный ключ
- Режим отладки

### Шаг 2: Запуск миграций базы данных

```bash
# Перейдите в директорию бэкенда
cd SiriusBackend

# Примените все миграции
python manage.py migrate
```

### Шаг 3: Создание суперпользователя (Опционально)

```bash
python manage.py createsuperuser
```

Следуйте инструкциям для создания админской учетной записи.

### Шаг 4: Запуск сервера разработки Django

```bash
python manage.py runserver
```

Бэкенд запустится на:
- **URL**: http://127.0.0.1:8000
- **Панель администратора**: http://127.0.0.1:8000/admin

### API эндпоинты бэкенда

- Аутентификация: `http://127.0.0.1:8000/api/auth/`
- Опросы: `http://127.0.0.1:8000/api/surveys/`
- Рекомендации: `http://127.0.0.1:8000/api/recommendations/`

## 🎨 Настройка и запуск фронтенда

### Шаг 1: Настройка переменных окружения

Создайте файл `.env` в корневой директории фронтенда:

```bash
cd SiriusFrontend/SiriusVue
```

Создайте файл `.env` со следующим содержимым:
```env
VITE_API_BASE_URL=http://127.0.0.1:8000/api
VITE_APP_TITLE=Sirius Investment Platform
```

Или скопируйте из примера:
```bash
cp .env.example .env
```

### Шаг 2: Запуск сервера разработки Vue.js

```bash
npm run dev
```

Фронтенд запустится на:
- **URL**: http://localhost:5173 (или другой порт, если 5173 занят)

### Возможности фронтенда

- Аутентификация (Вход/Регистрация)
- Инвестиционный опрос
- Результаты профиля риска
- Инвестиционные рекомендации
- Профиль пользователя

## 💻 Запуск обоих серверов (2 консоли)

### Полный процесс настройки

#### Консоль 1: Сервер бэкенда

1. **Откройте первую консоль/терминал**

2. **Перейдите в директорию бэкенда:**
   ```bash
   cd SiriusBackend
   ```

3. **Активируйте виртуальное окружение:**
   ```bash
   # Если используете Pipenv:
   pipenv shell
   
   # ИЛИ если используете venv на Windows:
   venv\Scripts\activate
   
   # ИЛИ если используете venv на Linux/Mac:
   source venv/bin/activate
   ```

4. **Запустите миграции (только первый раз):**
   ```bash
   python manage.py migrate
   ```

5. **Запустите сервер Django:**
   ```bash
   python manage.py runserver
   ```

6. **Оставьте эту консоль открытой** - Вы должны увидеть:
   ```
   Watching for file changes with StatReloader
   Performing system checks...
   
   System check identified no issues (0 silenced).
   January 27, 2026 - 10:00:00
   Django version 4.x, using settings 'SiriusMain.settings'
   Starting development server at http://127.0.0.1:8000/
   Quit the server with CONTROL-C.
   ```

#### Консоль 2: Сервер фронтенда

1. **Откройте вторую консоль/терминал** (первую консоль оставьте запущенной)

2. **Перейдите в директорию фронтенда:**
   ```bash
   cd SiriusFrontend/SiriusVue
   ```

3. **Установите зависимости (только первый раз):**
   ```bash
   npm install
   ```

4. **Запустите сервер разработки Vue.js:**
   ```bash
   npm run dev
   ```

5. **Оставьте эту консоль открытой** - Вы должны увидеть:
   ```
     VITE v5.x.x  ready in xxx ms
   
     ➜  Local:   http://localhost:5173/
     ➜  Network: use --host to expose
     ➜  press h + enter to show help
   ```

### Доступ к приложению

Когда оба сервера запущены:

1. **Откройте браузер** и перейдите по адресу: http://localhost:5173

2. **Фронтенд будет общаться с бэкендом** по адресу http://127.0.0.1:8000

3. **Теперь вы можете:**
   - Зарегистрировать новую учетную запись
   - Войти в систему
   - Пройти инвестиционный опрос
   - Просмотреть свой профиль риска и рекомендации
   - Управлять своим профилем

### Остановка серверов

Для остановки серверов:
- В каждой консоли нажмите `Ctrl + C`
- Закройте окна консоли, когда закончите

## 🔍 Решение проблем

### Проблемы с бэкендом

**Проблема: Порт 8000 уже занят**
```bash
# Используйте другой порт
python manage.py runserver 8001
```

**Проблема: Ошибки миграций**
```bash
# Сбросьте миграции (ПРЕДУПРЕЖДЕНИЕ: Это удалит данные)
python manage.py migrate --fake-initial
python manage.py migrate
```

**Проблема: Модуль не найден**
```bash
# Переустановите зависимости
pipenv install
# или
pip install -r requirements.txt
```

### Проблемы с фронтендом

**Проблема: Порт 5173 уже занят**
```bash
# Используйте другой порт
npm run dev -- --port 5174
```

**Проблема: npm install не работает**
```bash
# Очистите кэш npm
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

**Проблема: Ошибки CORS**
- Убедитесь, что настройки CORS на бэкенде разрешают URL фронтенда
- Проверьте конфигурацию CORS в [`SiriusBackend/SiriusMain/settings.py`](SiriusBackend/SiriusMain/settings.py)

### Проблемы с подключением

**Проблема: Фронтенд не может подключиться к бэкенду**
1. Проверьте, запущен ли сервер бэкенда
2. Проверьте URL API в файле `.env` фронтенда
3. Проверьте консоль браузера на наличие сообщений об ошибках
4. Убедитесь, что CORS правильно настроен на бэкенде

**Проблема: База данных заблокирована**
```bash
# Удалите файл базы данных (SQLite) и снова запустите миграции
# ПРЕДУПРЕЖДЕНИЕ: Это удалит все данные
rm db.sqlite3
python manage.py migrate
```

## 📝 Дополнительные заметки

### Режим разработки
- Оба сервера работают в режиме разработки с горячей перезагрузкой
- Изменения в коде автоматически перезапустят серверы
- Режим отладки включен для фронтенда и бэкенда

### Развертывание в продакшн
Для развертывания в продакшне вам нужно:
- Установить `DEBUG = False` в настройках Django
- Использовать WSGI сервер для продакшна (Gunicorn, uWSGI)
- Собрать фронтенд с помощью `npm run build`
- Разместить собранный фронтенд на веб-сервере (Nginx, Apache)
- Настроить правильные CORS и параметры безопасности
- Использовать переменные окружения для конфиденциальных данных

### База данных
- По умолчанию: SQLite (хорошо для разработки)
- Для продакшна: Рассмотрите PostgreSQL или MySQL
- Обновите настройки базы данных в [`SiriusBackend/SiriusMain/settings.py`](SiriusBackend/SiriusMain/settings.py)

### Документация API
- API Django REST Framework: http://127.0.0.1:8000/api/
- Панель администратора: http://127.0.0.1:8000/admin/
